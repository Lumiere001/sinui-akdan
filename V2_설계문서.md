# 신의 악단 V2 설계 문서

## 1. 변경 개요

V1에서 V2로의 주요 변경사항:
- 서약서 기능 추가 (최초 1회, 서버 저장)
- 팀 대표 ↔ 관리자 채팅 기능 추가
- Stage 2 재설계: 1곳 찾기 → 3단계 순차 장소 탐색
- 팀별 개별 30분 타이머
- 관리자 대시보드 강화 (지도, 채팅, 타이머 통합)
- 데이터 영속성: JSON 파일 저장

## 2. 확정된 설계 결정

| 항목 | 결정 |
|------|------|
| 데이터 저장 | JSON 파일 (추가 비용 없음) |
| 팀 대표 | 팀당 1명만, 서버에 저장, 재접속 시 복원 |
| 오답 처리 | 바로 다른 후보 장소로 이동 가능 |
| 장소 수 | 기존 8곳 유지, 팀별 방문 순서만 다르게 |
| 라운드 | 팀 1-5 함께 진행, 이후 팀 6-10 함께 진행 |
| 타이머 | 팀별 개별 30분, 관리자가 팀별로 시작 |
| 타이머 만료 | 시간 초과 알림 + 현재까지 진행 결과 표시 (참여자/관리자 모두) |
| 장소 해금 조건 | 해당 장소에 팀원 3명 이상 도착 |
| 후보 장소 거리 | 서로 300~400m 이내 |

## 3. 전체 사용 흐름

### 3-1. 참여자 흐름

1. 웹사이트 접속 → 팀 번호 + 비밀번호 + "팀 대표입니다" 체크박스
2. 서버가 서약서 완료 여부 확인
   - 완료: 서약서 건너뛰고 대기/게임 화면으로
   - 미완료: 서약서 화면으로 이동
3. 서약서 작성 (최초 1회) → 서버에 완료 기록 저장
4. 대기 화면 (관리자가 타이머 시작할 때까지)
   - 팀 대표만 채팅 가능
5. Stage 2 시작 (관리자가 해당 팀 타이머 시작):
   - 30분 타이머 가동
   - 지도에 팀원 위치 실시간 표시
   - 1단계: 서사적 힌트 + 후보 2곳 → 3명 이상 도착 시 해금 → 정답이면 2단계 / 오답이면 오답 사진 후 재시도
   - 2단계: 새 힌트 + 새 후보 2곳 → 같은 방식
   - 3단계: 새 힌트 + 새 후보 2곳 → 같은 방식
6. 3단계 모두 정답: "CCC 센터로 오세요" + 정답 악보 사진
7. 타이머 만료: 시간 초과 알림 + 현재까지 진행 결과

### 3-2. 관리자 흐름

1. 관리자 로그인 → 관리 페이지
2. 대시보드에서 한 눈에:
   - 각 팀 상태 (서약서 완료 인원, 현재 단계, 타이머 남은 시간)
   - 지도에서 각 팀별 인원 위치 실시간 확인
   - 모든 팀 채팅을 한 화면에서 관리
3. 팀별 타이머 개별 시작
4. 실시간 모니터링 + 채팅 응답

### 3-3. 재접속 흐름

1. 팀 번호 + 비밀번호 + 대표 여부 입력
2. 서버가 playerId(기기 고유 ID)로 사용자 식별
3. 서약서 완료 여부 → 완료했으면 건너뜀
4. 대표 여부 → 이전에 대표였으면 자동 복원
5. 게임 진행 상태 복원 (현재 단계, 타이머 남은 시간 등)

### 3-4. 채팅 (상시)

- 팀 대표 1명 ↔ 관리자 간 1:1
- 서약서 작성 후부터 이용 가능
- 관리자는 모든 팀 채팅을 한 화면에서 동시 관리

## 4. Stage 2 상세 설계

### 4-1. 장소 경로 구성

기존 8개 장소를 활용하여 각 팀별 3단계 경로를 구성한다.
각 단계의 후보 2곳은 서로 300~400m 이내여야 한다.

팀마다 같은 장소를 방문하더라도 순서가 다르다:
- 예: A팀 1→2→3, B팀 3→1→2

## 5. 기술 구현 계획

### 5-1. 데이터 영속성 레이어

- JSON 파일로 서버 데이터 주기적 저장
- 서버 시작 시 JSON 파일에서 복원
- 저장 항목: 서약서 완료 목록, 팀 대표 정보, 채팅 내역, 게임 진행 상태, 타이머 상태

### 5-2. 수정 필요 파일

서버:
- server/index.ts — 소켓 이벤트 전면 재설계
- server/gameData.ts — 3단계 순차 경로 데이터
- server/gameState.ts — 팀별 개별 타이머, 단계 진행 상태
- server/shared/types.ts — 새 이벤트 타입 정의
- server/persistence.ts (신규) — JSON 파일 저장/복원
- server/adminRoutes.ts — 팀별 타이머 시작 API

클라이언트:
- shared/types.ts — 새 이벤트 타입 정의
- client/src/pages/Landing.tsx — 대표 체크박스 추가
- client/src/pages/Pledge.tsx (신규) — 서약서 페이지
- client/src/pages/Game.tsx — 3단계 순차 진행 UI
- client/src/pages/Result.tsx — 단계별 결과/최종 결과
- client/src/pages/Admin.tsx — 대시보드 전면 재설계
- client/src/components/Chat.tsx (신규) — 채팅 컴포넌트

### 5-3. 새 소켓 이벤트

클라이언트 → 서버:
- pledge:submit — 서약서 제출
- chat:send — 채팅 메시지 전송
- admin:startTimer — 관리자가 특정 팀 타이머 시작

서버 → 클라이언트:
- pledge:status — 서약서 완료 여부
- chat:message — 채팅 메시지 수신
- chat:history — 채팅 내역 (재접속 시)
- team:stageUpdate — 단계 변경 알림
- team:timerStart — 타이머 시작
- team:timerExpired — 타이머 만료
- team:complete — 3단계 모두 완료
